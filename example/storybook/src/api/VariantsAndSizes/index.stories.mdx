# Variants & Sizes

- Dank allows you to easily create different variations of a component with different styles.
- Using variant props in a UI component library allows for more flexibility and reusability of components.

- Adding a variant
  - You can add a new variant by passing `variants` key in `styled`. There are no restrictions on the number of variations that may be incorporated.
  ```jsx
  const StyledButtonVariant = styled(Pressable, {
    p: '$3',
    variants: {
      variant: {
        solid: {
          bg: '$primary600',
      },
    },
  });
  ```

    .
- Multiple variants
  ```jsx
  const StyledButtonVariant = styled(Pressable, {
    p: '$3',
    variants: {
      solid: {
        bg: '$primary600',
      },
      subtle: {
        bg: '$primary200',
      },
    },
  });
  ```
- Default variant

  - You can pass the **`variant`** prop in the **`defaultProps`** property when defining the component to set a variant as the default.

  ```jsx
  const StyledButtonVariant = styled(Pressable, {
    p: '$3',
    variants: {
      variant: {
        solid: {
          bg: '$primary600',
        },
        subtle: {
          bg: '$primary200',
        },
      },
    },
    defaultProps: {
      variant: 'solid',
    },
  });
  ```

## Sizes

- Sizes work similarly to variants in that they allow you to create different variations of a component with different styles based on specific conditions. However, instead of changing the visual appearance of the component, sizes are used to change the layout of the component.

```jsx
const StyledButton = styled(Pressable, {
  variant: {
    size: {
      xs: {
        px: '$3',
        py: '$2',
      },
      sm: {
        px: '$3',
        py: '$2',
      },
      md: {
        px: '$3',
        py: '$2.5',
      },
      lg: {
        px: '$3',
        py: '$3',
      },
    },
  },
  defaultProps: {
    size: 'md',
  },
});
```

Similar to variants, you can also define default sizes by using defaultProps and set default value for **`size`** property.
